<h2 style="font-size:28px; font-weight:700;" align="center">üß© How the System Works</h2>

<h3 style="font-size:22px; font-weight:700;">1Ô∏è‚É£ Understanding the Data</h3>
<p style="font-size:16px; line-height:1.7;">
The system is built using real property listings that include details such as area (in sqft), number of bedrooms, bathrooms, furnishing type, sector/location, and the actual price.  
This forms the foundation of the entire project ‚Äî the model learns how price changes with the combination of these features.
</p>

<h3 style="font-size:22px; font-weight:700;">2Ô∏è‚É£ Cleaning & Preparing the Data</h3>
<p style="font-size:16px; line-height:1.7;">
Real-estate data is never clean when you first get it.  
Some entries may be missing important details, some may contain unrealistic values, and a few may even be formatted incorrectly.

To make sure the model learns correctly, the dataset goes through several steps:
</p>

<ul style="font-size:16px; line-height:1.7;">
<li>Fixing or filling missing values</li>
<li>Removing obviously wrong entries or extreme outliers</li>
<li>Standardizing formats such as area and price</li>
<li>Converting location and text-based fields into numerical form</li>
</ul>

<p style="font-size:16px; line-height:1.7;">
After this preparation, the data becomes structured and reliable ‚Äî ready for machine learning.
</p>

<h3 style="font-size:22px; font-weight:700;">3Ô∏è‚É£ Creating Useful Features</h3>
<p style="font-size:16px; line-height:1.7;">
The system doesn‚Äôt rely only on raw data.  
It creates additional helpful features that make patterns clearer, such as:
</p>

<ul style="font-size:16px; line-height:1.7;">
<li><b>Price per square foot</b> (helps compare properties fairly)</li>
<li><b>Location-based scoring</b> (premium vs budget areas)</li>
<li><b>Structured text features</b> extracted from property descriptions</li>
</ul>

<p style="font-size:16px; line-height:1.7;">
These features help the model understand the deeper relationships between area, location, rooms, and pricing.
</p>

<h3 style="font-size:22px; font-weight:700;">4Ô∏è‚É£ Training the Machine Learning Model</h3>
<p style="font-size:16px; line-height:1.7;">
Once the data is prepared, the model is trained to recognize patterns ‚Äî  
for example, how much a 2BHK in Sector 57 usually costs, or how prices increase with extra square footage.

Algorithms like Linear Regression and Random Forest are used because they work well with real-estate pricing, where the relationships are sometimes simple and sometimes very non-linear.
</p>

<p style="font-size:16px; line-height:1.7;">
The final model is the one that predicts prices most accurately across different locations and property types.
</p>

<h3 style="font-size:22px; font-weight:700;">5Ô∏è‚É£ How Price Prediction Happens</h3>
<p style="font-size:16px; line-height:1.7;">
When a user enters details like BHK, area, and sector, the app converts the inputs into the same feature format used during training.  
These features are fed into the model, and it instantly returns an estimated price based on everything it has learned.
</p>

<h3 style="font-size:22px; font-weight:700;">6Ô∏è‚É£ How Apartment Recommendations Work</h3>
<p style="font-size:16px; line-height:1.7;">
The recommendation system works in two simple steps:
</p>

<ol style="font-size:16px; line-height:1.7;">
<li><b>Location filtering:</b>  
   First, the system finds all apartments within the distance selected by the user.  
   This allows users to find properties near a specific landmark, office, or metro station.
</li>

<li><b>Similarity matching:</b>  
   Then, the system compares apartments based on their features (price, area, BHK, locality, etc.) and recommends the ones that are most similar.  
   This helps users discover alternatives they might have missed.
</li>
</ol>

<p style="font-size:16px; line-height:1.7;">
Together, these two steps create a personalized shortlist of properties tailored to the user's preferences and location.
</p>
